# open62541-compat — deep-init briefing

_Generated by deep-init on 2026-06-09, at repo state cb96eca (v1.4.7)._

## A. Summary

open62541-compat is a C++ adapter library that exposes the commercial Unified Automation SDK (UA-SDK) C++ API, backed by the open-source open62541 OPC UA stack (C99). It exists so CERN's quasar framework — and independent UA-SDK-style applications — can build OPC UA servers/clients on a free stack without rewriting UA-SDK-targeting code (`README.md:2`). The repo bundles an amalgamated open62541 v1.2.2 (`extern/open62541/`, stamped in `extern/open62541/include/open62541.h:3`); own code is ~5k lines wrapping ~101k vendored lines. Maintained by the quasar-team GitHub org; current release v1.4.7 (= HEAD, 2026-06-09); work tracked via CERN JIRA `OPCUA-*` tickets. Two consumption modes: quasar-embedded OBJECT library (default) or standalone static/shared library.

## C. Run / build / test / lint

Standalone (what CI and the local `build/` use):

```bash
cmake -B build -DSTANDALONE_BUILD=ON          # add -DOPEN62541-COMPAT_BUILD_CONFIG_FILE=boost_standard_install.cmake if find_package(Boost) needs help
cmake --build build -j$(nproc)
./build/test/open62541-compat-Test            # unit tests (gtest binary)
```

Variants: `-DSTANDALONE_BUILD_SHARED=ON` (shared lib), `-DSKIP_TESTS=ON`, `-DSERVERCONFIG_LOADER=ON` (needs `xsdcxx`). CI (`.github/workflows/ci.yml:27`) configures with `-DSTANDALONE_BUILD=ON -DCMAKE_CXX_FLAGS=-Wno-error=maybe-uninitialized` on AlmaLinux 9/10 containers. Quasar-embedded mode is the default (no `STANDALONE_BUILD`): produces OBJECT library `open62541-compat` plus the `quasar_opcua_backend_is_ready` target (`CMakeLists.txt:108-114`) and is driven from a quasar checkout (see quasar's `Documentation/AlternativeBackends.html`). Windows: VS2017 generator with `boost_custom_win_VS2017.cmake` and `BOOST_PATH_HEADERS`/`BOOST_PATH_LIBS` env vars (`README.md:59-79`). There is no linter/formatter config; the vendored C is compiled with a long warning list but deliberately not `-Werror` (`CMakeLists.txt:85`, commit b984601).

## D. Architecture

Three layers. (1) Vendored open62541 C amalgamation compiled into the library (`CMakeLists.txt:82-86`); optional `PULL_OPEN62541=ON` fetches/builds upstream instead ("not recommended", `CMakeLists.txt:26`). (2) Value-type wrappers, each owning a `UA_*` impl with deep-copy semantics via `UA_*_copy` and an `impl()` accessor: `UaString`, `UaNodeId`, `UaVariant` (the largest, `src/uavariant.cpp` — boost `numeric_cast` conversions, 1-D arrays only), `UaDataValue` (mutex-guarded for concurrent publish/read, `include/uadatavalue.h:36`), `UaDateTime`, `UaByteString`, `UaStatus`. (3a) Server façade: `UaServer` (wraps `UA_Server`) + `NodeManagerBase` (address-space construction) + node classes (`UaNode`, `OpcUa::BaseObjectType`, `OpcUa::BaseDataVariableType`, `OpcUa::BaseMethod`, `UaPropertyCache`, `UaPropertyMethodArgument`). (3b) Client façade: `UaClientSdk::UaSession`. The C↔C++ glue is three static callbacks — `unifiedRead`/`unifiedWrite`/`unifiedCall` (`src/nodemanagerbase.cpp:64-194`) — that recover C++ objects from open62541 `nodeContext`/`methodContext` pointers. `include/stack_fake_structures.h`, `other.h`, `session.h` supply minimal/empty stand-ins for UA-SDK types quasar references but this layer doesn't implement (`Session`, `ServiceContext`, `UaMutexRefCounted`, `DiagnosticInfo`). Arrays: `UaCompatArray<T>` (std::vector-backed, `.at()` bounds-checked, `include/array_templates.h`) and `ManagedUaArray<T>` RAII over `UA_Array` (`include/managed_uaarray.h`). Logging: `src/logit_logger.cpp` adapts `UA_Logger` to LogIt component `"open62541"`.

## E. Entrypoints and execution flow

The library has no `main`. Server lifecycle (driven by a quasar app): construct `NodeManagerBase` + `UaServer`; `linkRunningFlag(&flag)` is mandatory; `UaServer::start()` (`src/uaserver.cpp:54`) creates `UA_Server` with `UA_ServerConfig_setMinimal` on port 4841 (or the port parsed from ServerConfig.xml when built with `SERVERCONFIG_LOADER`, `src/uaserver.cpp:109-151`), installs the LogIt logger, calls `m_nodeManager->linkServer()` + `afterStartUp()` — where quasar-generated code populates the address space through `addNodeAndReference` → `UA_Server_add{Object,DataSourceVariable,Variable,Method}Node` — then spawns `runThread()` looping `UA_Server_run_iterate` while `*m_runningFlag`. Runtime data flow: OPC UA write → `unifiedWrite` → `BaseDataVariableType::setValue` (access-mask check, `src/opcua_basedatavariabletype.cpp:48`); read → `unifiedRead` → `value()` → `m_currentValue.clone()`; method call → `unifiedCall` → `MethodHandleUaNode` → `BaseObjectType::beginCall` (overridden by quasar device classes) with a `SynchronousMethodCallback`. Client flow: `UaSession::connect/read/write/call/browse` (`src/uaclient/uasession.cpp`). Test entrypoint: `test/src/main.cpp` (gtest). Example: `examples/src/ex_simple_client_read.cpp`.

Runtime: endpoint port is hardcoded 4841 unless `SERVERCONFIG_LOADER` is compiled in and a ServerConfig.xml is loaded, whose `UaEndpoint/Url` must match `opc.tcp://[NodeName]:<port>` exactly (`src/uaserver.cpp:141-149`); exactly one endpoint supported; the XSD mirrors UA-SDK's full ServerConfig shape but the loader extracts only the port.

## H. Testing strategy

Unit tests only, googletest-based, built only in standalone mode — explicitly deferred for quasar-embedded builds "until the quasar framework has a comprehensive strategy for testing optional modules" (`CMakeLists.txt:185-189`). Coverage targets the value-type layer: variant conversions including out-of-range rejection via `boost::numeric_cast` (`test/src/uavariant_test.cpp`), arrays, bytestring, datavalue, datetime, qualifiedname. The server (`NodeManagerBase`, `UaServer`) and client (`UaSession`) layers have no in-repo tests — they need a live stack. No coverage tooling, no sanitizers in CI. CI runs the suite on AlmaLinux 9 and 10 in one configuration (static, bundled open62541, LogIt-as-source) (`.github/workflows/ci.yml:33`). `README.md:82-97` sets the cultural expectation: new features come with tests ("feel guilty" if you skip them). Known gap: `UaVariant::operator==` is only tested for strings (`test/src/uavariant_test.cpp:176`).

## I. Coding conventions / patterns observed

- **UA-SDK fidelity beats C++ taste**: public classes, methods, and header names replicate UA-SDK (`include/uaclient/uaclientsdk.h`, `setByteString(value, detach)`…). Don't "modernize" signatures — quasar-generated code compiles against them.
- Wrappers own a `UA_*` impl, expose `impl()` (sometimes `pimpl()`), deep-copy via `UA_*_copy`, destroy via `UA_*_clear`/`_delete`.
- Failure handling: `OPEN62541_COMPAT_LOG_AND_THROW(type, msg)` macro (`include/open62541_compat.h:50`) or `alloc_error` (`include/open62541_compat_common.h`). Unimplemented UA-SDK surface throws loudly instead of faking success (e.g. detach branches in `src/uavariant.cpp`, `addUaReference` at `src/nodemanagerbase.cpp:488`).
- Logging via LogIt `LOG(Log::LVL)`; TRC for per-call chatter; component `"open62541"` for stack-internal logs.
- License headers: LGPL-3.0 "part of Quasar" on C++ sources; BSD-style on CMake scripts.
- Commit messages and branches carry `OPCUA-NNNN` JIRA keys; changes land via GitHub PRs on `master`.
- Mixed tabs/spaces, no `.clang-format` — match the file you're editing.

## J. Risks, oddities, technical debt

- **`UaVariant::operator==` scalar/array comparison looks wrong** (`src/uavariant.cpp:145`): `memcmp` length is `arrayLength` — 0 for scalars (same-type scalars would always compare equal) and element-count-not-bytes for arrays. Only the string path is tested. `UaDataValue::operator==` delegates to it (`src/uadatavalue.cpp:75`). Verify before relying on either.
- **Configure-time network dependencies**: googletest is git-cloned during `cmake` configure (`test/CMakeLists.txt:9`), LogIt is fetched at configure (`LogIt.cmake`). Offline builds need pre-populated trees (historical `OPCUA-1293_build_without_internet_connectivity` branches).
- **Stale docs/badges**: `doc/ChangeLog.html` ends at 1.4.3 (Nov 2022) though v1.4.7 is tagged; README badges point to dead travis-ci.org and Ben's appveyor while real CI is GitHub Actions.
- **Probable variable-name slip**: `test/CMakeLists.txt:33` links `${LOGIT_LIB}` but the root file sets `LOGITLIB` (`CMakeLists.txt:178`) — harmless in default LogIt-as-source mode, likely breaks test linking with external LogIt.
- **Deliberate startup leaks / ownership quirks**: `make_localised` leaks a `UaString` per call (`src/nodemanagerbase.cpp:199-205`); `inArgs`/`outArgs` arrays are never freed (`src/nodemanagerbase.cpp:381,391`); `UaServer::stop()` deletes the NodeManager it was merely handed (`src/uaserver.cpp:157`).
- **Hardcoded limits**: namespace index must be 2 (`src/nodemanagerbase.cpp:497`), one NodeManager (`src/uaserver.cpp:101`), one endpoint, client read/write of exactly one node per call (`src/uaclient/uasession.cpp:176,262`), browse is forward-only (`uasession.cpp:400`).
- `OPCUA_2603` client keep-alive thread exists but is compiled out (no `-DOPCUA_2603`). 46 TODO/FIXME markers in own code. `getNode` is an O(n) list scan (`src/nodemanagerbase.cpp:52`). `include/opcua_platformdefs.h` lacks an include guard (benign — typedefs only). `cmake/Open62541CompatFetchContent.cmake` predates CMake 3.11's FetchContent and could be retired now that the minimum is 3.16.

## K. Pre-change checklist

1. **Never reshape public headers in `include/`** — they mirror UA-SDK; quasar-generated code depends on exact signatures (`README.md:2`).
2. **Build both modes** before pushing: standalone (`-DSTANDALONE_BUILD=ON`) *and* the default quasar OBJECT-library mode (`CMakeLists.txt:108`) — flags and include paths differ between them.
3. **Run `./build/test/open62541-compat-Test`** and add tests under `test/src/` for any value-type behavior you add or change.
4. **Never hand-edit `extern/open62541/*`** — regenerate via `extern/open62541/prepare_open62541.sh` (bump `TAG`), committing with the `open62541` author convention the script applies.
5. **Keep the nodeContext invariants**: whatever you pass to `UA_Server_add*Node` as context must be a `BaseDataVariableType*` / `MethodHandleUaNode*` — the static callbacks `static_cast` blindly (`src/nodemanagerbase.cpp:78,107,161`).
6. **Namespace index 2 is load-bearing** (`src/nodemanagerbase.cpp:497-499`, `getNameSpaceIndex()` at `include/nodemanagerbase.h:51`); changing it breaks quasar address-space assumptions.
7. **New `UaVariant` type support** means touching together: `OpcUaType` enum (`include/uavariant.h:34`), `convertNumericType`/`isNumericType` (`src/uavariant.cpp`), and tests.
8. **Don't add `-Werror` to the amalgamation C flags** (`CMakeLists.txt:85`) — deliberately removed in OPCUA-3384 (commit b984601).
9. **Throw on unimplemented paths** with `OPEN62541_COMPAT_LOG_AND_THROW`; never silently no-op.
10. **Windows stays supported**: mind `NOMINMAX` (`CMakeLists.txt:94`), `ws2_32`, and the VS2017 toolchain files when touching build logic.
